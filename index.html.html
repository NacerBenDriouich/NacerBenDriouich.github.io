Supply Chain & Sales Analysis

A full BI solution built on transactional retail data — from raw SQL extraction to an interactive executive dashboard.

Business Request

A bike shop needed a Power BI report to analyze sales performance, product trends, and customer behavior across regions and time periods — with enough interactivity to support both operational and strategic decisions.

Approach
  • Data Extraction: Pulled from SQL database — calendar, customer, product, and sales fact tables
  • Data Modeling: Built a star schema with a central sales fact table and four dimension tables; cleansed using SQL (deduplication, type validation, outlier removal)
  • DAX Measures: ROI, Profit Margin, Customer Lifetime Value, revenue growth by period
  • Dashboard: Interactive Power BI with slicers, drill-through, custom visuals, and KPI cards
Key Findings
  • Mountain Bikes drove 95% of total revenue ($21M) — accessories and clothing significantly underperformed
  • Sales peaked in May and December — inventory and staffing cycles were misaligned
  • East Coast cities (Washington DC) and West Coast (Seattle, San Francisco) accounted for the highest sales density
SQL Sample
SELECT c.customerkey, c.firstname + ' ' + c.lastname AS [Full Name], CASE c.gender WHEN 'M' THEN 'Male' WHEN 'F' THEN 'Female' END AS Gender, c.datefirstpurchase, g.city AS [Customer City] FROM [AdventureWorksDW2019].[dbo].[DimCustomer] AS c LEFT JOIN dbo.dimgeography AS g ON g.geographykey = c.geographykey ORDER BY CustomerKey ASC

Olympic Games Historical Analysis

Interactive historical performance dashboard for Summer Olympic Games — built for a media outlet use case.

Business Request

A news media outlet needed to help readers understand how countries have historically performed in the Summer Olympics, with the ability to filter by country, athlete, sport, and medal type.

Data Transformation (SQL)
SELECT [ID], [Name] AS 'Competitor Name', CASE WHEN SEX = 'M' THEN 'Male' ELSE 'Female' END AS Sex, [Age], CASE WHEN [Age] < 18 THEN 'Under 18' WHEN [Age] BETWEEN 18 AND 25 THEN '18-25' WHEN [Age] BETWEEN 25 AND 30 THEN '25-30' WHEN [Age] > 30 THEN 'Over 30' END AS [Age Grouping], LEFT(Games, CHARINDEX(' ', Games) - 1) AS 'Year', [Sport], [Event], CASE WHEN Medal = 'NA' THEN 'Not Registered' ELSE Medal END AS Medal FROM [olympic_games].[dbo].[athletes_event_results] WHERE RIGHT(Games, CHARINDEX(' ', REVERSE(Games))-1) = 'Summer'
Features
  • Filter by country, athlete, sport, and medal type
  • Age group segmentation across competitors
  • Historical medal trends over time by nation

Supplier List Matching Tool

Automated fuzzy-matching tool to reconcile Amazon's public supplier list against an internal SQL database — handling real-world inconsistencies at scale.

The Problem

The business needed to match a large Excel file of facilities (from Amazon's public supply chain disclosure) against an internal SQL Server database. Because records used inconsistent casing, abbreviations, and spelling variations, exact matching failed. Manual lookups were taking hours per batch.

Example mismatches: "PT. SEMARANG GARMENT" vs "Pt Semarang Garment" — same company, impossible to match exactly.

Approach
  • Built with Python (Pandas, RapidFuzz, pyodbc)
  • Normalized both datasets: lowercased, trimmed whitespace, collapsed extra spaces
  • Applied fuzzy string matching (token_set_ratio) on company names
  • Added a secondary address similarity score to validate matches
  • Exported combined Excel file with original data, best match, name score, and address score
  • Results capped at 80% threshold — anything below flagged for manual review
Impact
Hours → MinutesManual record lookup time eliminated through automated matching pipeline
RepeatableTool runs on any new supplier list submission without modification
AccurateHandles case, spacing, and naming variation that breaks exact matching
AuditableEvery match includes a confidence score for review and validation

Certificate Automation Pipeline

Eliminated a manual, hours-consuming certification process at a global compliance organization — reducing per-certificate time from 30–45 minutes to under 1 minute.

The Problem

Compliance certificates were produced entirely by hand — staff would populate a Word document field by field, format it manually, convert it to PDF, and email or FedEx it to facilities worldwide. For a global organization certifying manufacturers across 170 countries, this was consuming significant staff hours and budget per certificate.

A secondary problem: international facility addresses were entered as free-form user input, arriving as a single unstructured line of text. This made records inconsistent, unverifiable, and unpresentable on certificates going to facilities in dozens of address formats.

Approach — Three Phases
  • Phase 1 (Semi-manual): Built structured data entry forms that auto-populated certificate template fields — reduced manual entry time significantly
  • Phase 2 (QR code): Introduced QR codes on certificates to enable digital verification and eliminate most physical FedEx mailing worldwide
  • Phase 3 (Full automation): One-click pipeline — pulls data from user submissions, generates the certificate with embedded QR code, converts to PDF, and sends automatically
  • Address standardization: Integrated Google Maps API to parse free-text address input into structured city, state/province, zip code, and country fields
Stack
  • Python (lxml, python-docx) for document generation
  • Word SDT content controls for dynamic field population
  • LibreOffice for PDF conversion
  • Google Maps API for address parsing and standardization
  • Power Automate for email delivery pipeline
